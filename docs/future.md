# Future Work

## iCloud 下载稳定性

- 多线程/并发下载：当前内置 `icloudpd==1.32.3` 的官方参数列表没有 `--max-concurrent-downloads`，因此不能直接把该参数写入运行命令。后续如果 `icloudpd` 官方支持并发下载参数，或项目改为自研下载层，再在后端和 Web 面板中加入可配置并发数。
- 下载延时/节流：当前内置 `icloudpd==1.32.3` 的官方参数列表没有 `--download-delay`，因此不能直接把该参数写入运行命令。后续如果上游支持下载间隔，或我们接管单文件下载循环，再加入“每个文件下载后等待 N 秒”的配置。
- 当前已实现：媒体同步失败后按配置自动重试，用于缓解 Apple 503、连接重置、临时网络抖动等导致的中途退出。
